[Tzimisce] | Discord Dicebot

[Tzimisce] is a Discord dicebot for Vampire: The Masquerade 20th Anniversary Edition and other White Wolf tabletop RPGs using the Storyteller dice system. It features simple syntax and a number of advanced features.

Storyteller Rolls

Basic Syntax

/m <pool> [difficulty] [specialty] # comment
  • pool (mandatory) is the number of dice to roll.
  • difficulty (optional, default 6) must be a number between 2 and 10. If it is outside those bounds, [Tzimisce] will automatically adjust the difficulty to the nearest bound (i.e. 11 will become 10 and 1 will become 2).
  • specialty (optional) will cause 10s to count as double successes when applied.
  • # comment (optional) is a label that will be applied to the roll. The # is required!
Examples
  • /m 5 — roll 5 dice at difficulty 6
  • /m 5 7 # Mask of 1000 Faces — 5 dice, difficulty 7, with a comment
  • /m 8 5 Domineering # Command — 8 dice, difficulty 5, specialty of "Domineering", and a comment

Using Willpower

/mw <syntax>

Adding Willpower to a roll adds an extra success which cannot be canceled by botches. The rest of the roll syntax is the same as above.

Disable Botches

/mz <syntax>

With this option set, botches are converted to simple failures.

Automatic Successes

/m <pool> <difficulty> <autos> [specialty] # comment

In order to add Potence-style automatic successes, both the pool and the difficulty must be supplied. Unlike Willpower, automatic successes may be canceled by botches. Willpower (above) and specialties may be used on this type of roll.

Compact Mode

/mw <syntax>

With this option set, the bot outputs using plain text instead of an embed.

Traditional Rolls

/m XdY+...+n # comment

[Tzimisce] can perform traditional rolls, such as 1d10+5, 2d10+3d6, etc. As of this writing, only addition is allowed.

Macros

Saving Rolls

/m <macro-name> = <syntax> # comment

The macro-name must not contain spaces but can contain both - and _ characters. The roll syntax can be any valid roll. If supplied, the # comment will be used every time you use the macro.

If the macro already exists, invoking this command will overwrite it. In this case, the previously stored comment will be retained unless a new one is supplied.

To use a macro, simply type /m <macro-name>. Willpower and a new comment may be used. Note: Saving a macro does not roll the macro!

Examples
  • /m attack = 7 Graceful # Dex + Brawl — Saves both the syntax and the comment
  • /m attack — Rolls the above macro
  • /m attack # Using claws — Rolls the macro but with a different comment
  • /m attack = 8 Graceful — Changes the roll of the above macro but does not touch the comment

Deleting Rolls

/m <macro-name> =

This will delete the macro.

/m $delete-all

This will delete all of your macros on the current server. Use with caution!

Modifying Comments

/m <macro-name> c= [new comment]

This will modify only the comment of the macro without touching the syntax. If left blank, it will delete the stored comment.

Modifying Macros

/m <macro-name> <pool-mod> [difficulty-mod]

This allows you to perform one-time modifications on a macro without saving a new one.

  • pool-mod must be positive (i.e. +2), negative, or zero. This will add or subtract dice from the stored pool.
  • difficulty will add or subtract from the stored difficulty (if a sign is supplied) or set a new difficulty (if a sign isn't supplied).
Examples
  • /m hunt 0 +1 — roll your hunt macro at +1 difficulty.
  • /m attack -1 — roll your attack macro with 1 fewer die.
  • /m command 0 5 — roll your command macro at difficulty 5.

Initiative

Rolling Initiative

/mi <mod>

Roll 1d10+mod under your display name and add it to the channel's initiative table. To roll under another character's name (NPCs, minions, etc.) write in the name after the mod.

Declaring Actions

/mi dec <action>

Declare an action for your character (your display name). This requires you to have already rolled initiative. To declare for a different character, add -n [character] to the end.

Other Commands

  • /mi remove [character] — Remove yourself (or [character], if provided) from the initiative table
  • /mi reroll — Reroll all initiative. This clears declared actions
  • /mi clear — Empty the initiative table

Customization

Changing Server Settings

/m set

By itself, displays the available parameters and their current values. Unless marked otherwise, each setting defaults to false.

  • use_compact — Always use compact mode for rolls
  • exploding_tens — When rolling a specialty, tens explode (roll an extra die, recursively) instead of counting as two successes
  • nullify_ones — When using the z roll option, ones do not subtract from successes
  • prefix — Change the bot's invocation prefix. Defaults to / and !

To change one of these settings: /m set <parameter> <new_value>.

Troubleshooting

Unless you are using compact mode (/mc), [Tzimisce] uses embeds to display roll results. If you cannot see these embeds, it may either be a server permissions issue or a user preferences issue. Ensure that the bot has permission to embed links under text channel permissions. Ensure users have "Show website preview info from links pasted into chat" is enabled under Text & Images.
